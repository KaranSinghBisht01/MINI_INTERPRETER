How It Works

Tokenizer (Lexer)

Scans source code character by character using regular expressions. Converts raw text into a structured list of tokens: NUMBER, OPERATOR, IF, ELSE, IDENTIFIER, etc.

Parser

Uses Precedence Climbing to parse tokens into an Abstract Syntax Tree (AST). Handles arithmetic, comparisons, if-else, assignments, and print statements.

IR Generator

Walks the AST and emits stack-based intermediate instructions: PUSH, LOAD, STORE, ADD, SUB, MUL, DIV, GT, LT, EQ, IF_FALSE, JUMP, PRINT, NEG.

Executor

Simulates a virtual stack machine. Maintains an operand stack, a symbol table for variables, and an instruction pointer for control flow (if-else, jumps).
